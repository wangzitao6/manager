# 班级管理模块设计文档（SDD）

## 1. 文档信息

- 模块名称：班级管理
- 文档类型：软件设计文档（SDD）
- 适用范围：后端接口开发、数据库设计、前端联调
- 技术栈：
    - Spring Boot
    - MyBatis
    - MySQL
    - Swagger / OpenAPI
    - Bean Validation
- 开发目标：实现班级信息的新增、删除、修改、查询能力

---

## 2. 背景与目标

### 2.1 背景
系统需要增加一个“班级”基础管理能力，用于维护班级档案数据，支持对班级信息进行统一管理。

### 2.2 目标
实现一个完整的班级管理模块，支持以下能力：

- 新增班级
- 删除班级
- 修改班级
- 查询班级详情
- 分页查询班级列表

### 2.3 设计原则
- 接口清晰，职责单一
- 数据结构简单可扩展
- 支持后续与学生、老师等模块关联
- 使用统一返回结构
- 支持参数校验和接口文档

---

## 3. 需求说明

### 3.1 功能需求

#### 3.1.1 新增班级
用户可创建一个新的班级，录入班级名称、年级、班主任、状态等信息。

#### 3.1.2 修改班级
用户可修改已存在的班级信息。

#### 3.1.3 删除班级
用户可根据班级 ID 删除班级。

#### 3.1.4 查询班级详情
用户可根据班级 ID 查询班级详细信息。

#### 3.1.5 分页查询班级列表
用户可分页查询班级列表，并支持按班级名称模糊搜索、按状态筛选。

---

## 4. 业务规则

### 4.1 班级唯一性
班级名称在同一个年级下建议保持唯一，避免重复录入。

### 4.2 删除规则
删除班级前需确认：

- 当前班级是否存在
- 若未来存在学生关联，可扩展为“逻辑删除”模式
- 当前版本可先支持物理删除，后续可升级为逻辑删除

### 4.3 状态规则
班级状态建议使用以下枚举：

- `ENABLED`：启用
- `DISABLED`：停用

### 4.4 字段校验规则
- 班级名称不能为空
- 班级名称长度不能超过 $$50$$
- 年级不能为空
- 状态不能为空
- 班主任姓名长度不能超过 $$50$$
